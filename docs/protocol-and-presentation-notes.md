# Protocol and Presentation Notes

## Why protocoling now
Capture implementation evidence continuously so presentation prep is assembly, not memory work.

## Minimal protocol schema
- Date
- Change summary
- Milestone link (`M1`..`M7`)
- Risk/assumption
- Validation evidence (test run, screenshots, manual scenario)

## Suggested recurring entries
- Business logic updates (rollup, top-down, status flow)
- UI behavior changes (tree, tooltips, sidebar, views)
- Integration updates (comparison import, export guard, adapters)
- Test coverage updates

## Demo script outline
1. Create plan with hierarchy and KPI.
2. Show list view tree, expand/collapse all.
3. Edit plan values and explain tooltip formulas.
4. Trigger top-down and show rounding consistency.
5. Move status `Draft -> InReview -> Approved`.
6. Show readonly behavior and export guard.
7. Import comparison data and highlight variances.
